## Project Description
GreenTech Solutions is a professional, high-end web platform designed for Small to Medium Businesses (SMBs). The site aims to inform and educate users on sustainability while driving conversions for eco-friendly hardware, software, and consulting services. The design is a hybrid of editorial warmth and tech-centric precision, featuring a massive, eccentric hero section and interactive product filtering.

## Technologies Used
* **HTML5:** Semantic structure for high web accessibility (WCAG compliant).
* **CSS3:** Custom properties (variables), Flexbox, and CSS Grid for a mobile-first responsive layout.
* **JavaScript (Vanilla):** Custom form validation, dynamic product filtering, and interactive navigation toggles.
* **Google Fonts:** Modern, clean typography (Segoe UI/Roboto).

## How to Set Up and Run
* **1. Clone the repository:** git clone https://github.com/karagfactor/KaraGFactor-web-project.git

* **2.Navigate to the directory:** Open the final folder.

* **3. Launch the site:** Open index.html in any modern web browser.

* **4. Directory Structure:** Ensure the images, styles.css, and script.js files remain in their respective directories for full functionality .

# Project 2 - Project 3 -- HTML Foundation Page
## Webpage Theme
The theme for this webpage is **"Beyond The Riverwalk: San Antonio Hidden Gems Guide."** This project serves as a digital extension of my short-term rental business, **Parkside Stay**. I designed the page with a **"Modern Boho-Luxe"** aesthetic, utilizing a soft cream and charcoal color palette to provide a professional, welcoming experience for guests looking to explore the lesser-known parts of San Antonio.

## What I Learned
During this project, I deepened my understanding of the following technical concepts:
* **Relative File Paths:** I learned how to navigate complex folder structures using `../../` to ensure images and links function correctly within a nested GitHub repository.
* **Document Hierarchy:** I practiced organizing content for better User Experience (UX) by implementing internal "Jump Links" to help users navigate long-form content on mobile devices.
* **Media Management:** I utilized the `object-fit: cover` property to maintain high-quality aspect ratios for circular thumbnail images without distorting the original files.
* **Accessibility Standards:** I ensured every image includes descriptive `alt` text and that external links open in new tabs to maintain site persistence.

## Challenges Faced
The most significant challenge was achieving a specific, high-end "branded" look while restricted to pure HTML. Without the use of external CSS, I had to get creative with table properties like `cellpadding`, `cellspacing`, and `border-radius` to create a clean, modern interface. I also had to troubleshoot several pathing issues on GitHub Pages to ensure my image assets were mapping correctly to the live site.
